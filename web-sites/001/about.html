Paul Ramnora Web Site

Introduction

I wanted to create a FREE web site; using a mixture of both Azure/GitHub cloud based technologies.

The hardware structure

  • The web site folders/pages are stored both...
    • locally, on my home computer
    • then each web site folder/page is uploaded/mirrored onto an external server/GitHub code repository called: test2
  • The web server -(which serves each page to the web browser when called/and, also, provides the base URL address)- is Microsoft Windows Azure platform which points to my GitHub code repository: test2.

The software structure

  1. The folder structure is built using Windows 10 operating system
  2. Each HTML web page is written using Windows 10 Notepad editor software
  3. When all of the folders/web pages are finished with being edited; I then use GitHub Desktop application to
    • Commit each folder/file to the stage
    • Upload/Sync between my home computer/and, the external GitHub server
  4. Finally, I open up a browser window, and, test all files are running, correctly by running them from Microsoft Azure external server URL address:
    http://pmrwebapp1.azurewebsites.net

NOTES

  • NOTE(1): GitHub, keeps a precise record of each change you make to any folders/files; which means, at any point in time, you can always see/revert back any wrong changes you might have made.

  • NOTE(2): The [index.html] web page is automatically served whenever one types in its base URL address; so, there is no need to call the file by it's full name:
    http://pmrwebapp1.azurewebsites.net/index.html
    ...instead, simply call the file using...
    http://pmrwebapp1.azurewebsites.net
  • NOTE(3): The [test2] folder is being pointed to 'invisibly' by Microsoft Windows Azure web server; which is why there was no need to mention 'test2' inside of the URL calls.
    http://pmrwebapp1.azurewebsites.net/test2/index.html
    ...instead, simply call the file using...
    http://pmrwebapp1.azurewebsites.net/index.html
  • NOTE(4): Getting Microsoft Windows Azure to point to a GitHub Repo does take some very careful setting up...; you have to know exactly where to look inside of the multiple menus in order to discover all of the right settings/options.

  • NOTE(5): Potentially, you can risk having to go face a huge bill by letting Windows Azure web server keep running, constantly, all the time: 24/7/366 non-stop! Therefore, whenever you are finished with 'testing' web pages; always remember to turn the server off by clicking option: 'stop'.

  • NOTE(6): You don't have to run web sites this way; there are, at least, 2 other possible options...

    • Run a FREE web site from inside of GitHub using either their online editor.
      http://pramnora.github.io

    • Run a FREE web site from inside of Windows Azure using their online editor.

    However, I find that developing/uploading web sites from home...is far easier, and, a much more truly comfortable way to work...without needing to be online all the time.
